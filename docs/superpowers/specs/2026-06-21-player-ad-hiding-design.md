# 播放器右下角广告隐藏设计

## 背景

`ikanbot-player-enhancer.user.js` 已通过一个 `MutationObserver` 适配播放器延迟加载和线路切换后的 DOM 重建。播放画面右下角可能出现包含 `60323.com` 等文字的广告覆盖层。本功能只隐藏该广告，保留字幕、页面左上角提示和右上角 `RTX HDR` 标识。

## 范围

- 只处理 `.video-js` 播放器内部的 DOM 广告覆盖层。
- 只匹配位于播放器右下区域且包含明确广告特征文本的元素。
- 支持首屏已有广告和后续动态插入的广告。
- 不裁剪、不模糊、不遮盖视频画面，不处理已经烧录进视频帧的水印。
- 不修改现有播放控制、快捷键和进度保存行为。

## 方案

采用“广告文本识别 + 位置约束 + DOM 隐藏”。固定 CSS 选择器容易随站点结构变化失效；固定遮罩会损失正常画面。内容和位置联合判断可以在结构变化时保持一定适应性，同时降低误隐藏字幕的风险。

## 组件与职责

### 广告文本识别

纯函数接收元素文本，规范化空白和大小写后判断是否包含已知广告域名 `60323.com`。域名是强特征，不使用“官网”“棋牌”等宽泛词单独命中，避免误判字幕。

### 右下区域判定

纯函数比较候选元素与播放器的矩形边界。候选元素的中心点必须位于播放器横向后 50% 且纵向后 50% 的区域，候选宽度和高度分别不得超过播放器的 45%，并且矩形具有正面积。无法取得有效几何信息时不隐藏。

### 广告容器定位

从命中文本的元素向上查找，但不越过播放器根节点。只要父元素仍满足文本、位置和最大尺寸条件，且不包含 `video` 或 `.vjs-control-bar`，就继续向上；最终选择满足这些条件的最外层容器。播放器根节点、视频元素和控制条永远不能成为隐藏目标。

### 隐藏与动态同步

隐藏函数仅在 `.video-js` 范围内扫描候选元素。确认目标后设置专用数据属性，并通过已有样式规则将目标设为 `display: none !important`。重复扫描是幂等的。

现有 `syncPlayer()` 在初始加载和 `MutationObserver` 通知后调用隐藏函数，因此线路切换或广告重新插入后会自动再次处理，不新增第二个观察器。

## 数据流

1. 脚本启动或播放器 DOM 发生变化。
2. `syncPlayer()` 获取所有 `.video-js` 播放器。
3. 在每个播放器中筛选包含 `60323.com` 的候选元素。
4. 检查候选元素是否位于播放器右下区域。
5. 定位安全的广告容器并标记隐藏。
6. 后续同步跳过已标记元素或保持相同标记，避免重复副作用。

## 安全与失败处理

- DOM 查询、文本读取或矩形读取异常时跳过当前候选，不影响播放器其他功能。
- 没有明确域名特征时不隐藏任何元素。
- 广告若属于跨域 iframe 内部或已经烧录进视频帧，脚本保持画面不变，不使用可能遮挡正常内容的降级遮罩。
- 广告节点隐藏后仍保留在 DOM 中，避免删除节点引发播放器脚本异常。

## 测试

按测试驱动方式覆盖以下行为：

- 域名文本能够识别，普通字幕和宽泛广告词不能识别。
- 只有播放器右下区域内的命中元素才符合隐藏条件。
- 容器定位不越过播放器根节点，也不会选择视频元素。
- 首次同步会标记广告容器。
- 重复同步不会产生额外变更。
- 没有广告或广告位于其他区域时，现有控制条和视频元素保持不变。

最后运行全部 Node 测试及 `node --check ikanbot-player-enhancer.user.js`。
