# ikanbot 播放器增强设计

## 范围

仅在 `https://www1.ikanbot.com/play/*` 运行的 Tampermonkey 用户脚本。脚本只操作页面现有 Video.js 播放器和浏览器本地存储，不读取或导出媒体地址，不抓取影片/线路，不解析令牌，也不发起网络请求。

## 交互

- 在 `.vjs-play-control` 后依次插入“后退 10 秒”和“前进 10 秒”按钮。
- `ArrowLeft` / `ArrowRight` 执行相同跳转；表单控件或可编辑区域聚焦时忽略。
- 跳转结果限制在 `0..duration`，并阻止方向键的页面默认行为。

## 进度

- 以 `/play/` 后的页面路径作为影片 ID，使用 `localStorage` 独立保存。
- 每 5 秒以及暂停、页面隐藏、离开页面时保存。
- 新播放器出现并获得元数据后恢复；不足 5 秒不恢复。
- 已结束或距片尾不足 30 秒时删除记录。
- 存储不可用、记录损坏、播放器状态暂不可读时静默降级，不影响播放。

## DOM 生命周期

使用 `MutationObserver` 侦测播放器延迟初始化和线路切换导致的 DOM 重建。按钮带稳定标记，重复同步不会重复插入。每个新 video 元素只绑定一次事件并尝试一次恢复；替换播放器前补存旧元素进度。

## 验证

使用 Node 内置 `node:test` 覆盖页面 ID、跳转边界、输入焦点过滤、保存/清除/恢复规则；另做 JavaScript 语法检查、元数据匹配检查和禁止网络 API 扫描。
